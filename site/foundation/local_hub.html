Stack Decision Matrix (Research-backed)

Recommended path: keep Ophtxn core control plane, then combine OpenClaw + remote worker layer, and use Perplexity Computer selectively for scoped research tasks.

Recommended

Ophtxn Core + OpenClaw + Orgo-style workers

  • Own UX, approvals, memory, and execution policy in Ophtxn.
  • Use OpenClaw for channel/runtime actions and plugin tools.
  • Run sub-agents on cheap remote machines when needed.
Selective

Ophtxn + Perplexity Computer

  • Use Perplexity Computer for deep browsing or connector-heavy tasks.
  • Keep final decisions and workflows routed back into Ophtxn.
  • Avoid platform lock-in by preserving first-party runbooks.
Not Ideal

Perplexity-only Control Plane

  • Faster start, weaker ownership of memory and governance.
  • Less flexibility for custom agent orchestration.
  • Higher long-term dependency risk for your product vision.

Hard rule: we do not remove model safety rails via jailbreaks. Ophtxn governance is where policy and permissions are tuned.

Skill Lanes

Use these lanes to keep skills unlocked but controlled.

Codex Lane

  • `playwright` for browser automation tasks.
  • `cloudflare-deploy` for deploy and domain operations.
  • `openai-docs` and `security-*` skills for reliable implementation.

OpenClaw Lane

  • Use channel configs + group policies for controlled agent actions.
  • Use OpenClaw Skills for repeatable social/research workflows.
  • Keep OpenClaw runtime as worker layer, not canonical memory.

Chrome Lane

  • Keep a dedicated "agent ops" profile with minimal extensions.
  • Pin only your run surfaces: Local Hub, Command Center, App Shell.
  • Use Perplexity tab/workspace only for bounded research sessions.
