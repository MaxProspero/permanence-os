Stack Decision Matrix

Research-backed · March 2026

✓ Recommended

Ophtxn Core + OpenClaw + Orgo Workers

  • Own memory, approvals, UX in Ophtxn.
  • OpenClaw handles channel/runtime execution.
  • Sub-agents run on cheap remote machines.
  • Full governance without vendor lock-in.
◎ Selective

Ophtxn + Perplexity Computer

  • Use for deep browsing + connector tasks.
  • Route results back into Ophtxn memory.
  • Keep runbooks in first-party docs.
  • Watch tiered limits — not unlimited.
✗ Not Ideal

Perplexity-only Control Plane

  • No ownership of memory or governance.
  • Less flexible for custom orchestration.
  • High platform dependency risk.
  • Conflicts with Ophtxn product identity.

Governance and policy routing replace jailbreak approaches — always.
