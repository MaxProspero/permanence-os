CLAUDE.md Setup

Tell your agent how your world works.

→ Explore

A CLAUDE.md file at your project root gives the agent persistent context — identity, file locations, coding standards, and guardrails. It’s read automatically at session start.

markdown# MY-PROJECT — CLAUDE.md ## Identity Owner: Your Name Repo: github.com/you/my-project ## Code Standards - Python: type hints on all public functions - Tests: pytest, 80%+ coverage target - Commits: feat(scope): description ## Guardrails - Never commit secrets or .env files - Run linter before every push

Tip: Keep it under 200 lines. Link to detailed docs instead of inlining everything.

MCP Integration

Connect Claude to external tools via protocol.

→ Explore

The Model Context Protocol lets Claude call external tools — databases, APIs, browsers. Define servers in your project’s .claude/settings.json.

json{ "mcpServers": { "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"] }, "notion": { "command": "npx", "args": ["-y", "notion-mcp-server"] } } }

Tip: Each server exposes tools Claude can call. Run /mcp to see active connections.

Hook System

Trigger automated workflows from agent events.

→ Explore

Hooks run shell commands before or after agent actions — linting on save, tests on commit, notifications on task complete. Defined in .claude/settings.json.

json{ "hooks": { "PreToolUse": [{ "matcher": "Bash", "hooks": [{ "type": "command", "command": "echo 'Reviewing…'" }] }], "PostToolUse": [{ "matcher": "Write", "hooks": [{ "type": "command", "command": "ruff check $CLAUDE_FILE" }] }] } }

Tip: Hook exit code 2 blocks the action. Use PreToolUse to enforce policy before writes.

Memory Persistence

Retain context and knowledge across sessions.

→ Explore

Give your agent durable memory by persisting structured context to files that get loaded at session start. Think of it as the agent’s notebook.

file treememory/ canon/ identity.md # who you are principles.md # operating rules working/ decisions.jsonl # logged choices session_state.json # resumable reflections/ weekly_review.md # agent self-eval

Tip: Use JSONL for append-only logs, Markdown for human-readable context. Load in CLAUDE.md bootstrap.

Advanced

Agent Governance

Policy enforcement + human approval gates.

→ Explore

Define risk tiers and approval gates so agents can act autonomously on safe tasks while escalating anything dangerous to a human.

pythonRISK_TIERS = { "low": {"auto": True}, "med": {"auto": True, "log": True}, "high": {"auto": False, "require": "human_approval"}, } def gate(action, tier): rule = RISK_TIERS[tier] if not rule["auto"]: return request_approval(action) return execute(action)

Tip: Default to no-spend, no-delete. Let the human unlock higher tiers explicitly.

Prompt Engineering

Get better outputs with better inputs.

→ Explore

Structure beats length. Give the agent role, context, constraints, and a concrete example of the output you want.

prompt✘ Vague: "Write me a good commit message" ✔ Structured: "Write a commit message for this diff. Format: feat(scope): 1-line summary Rules: - Under 72 chars - Imperative mood - No period at end Example: feat(auth): add JWT refresh"

Tip: One concrete example is worth 100 words of description. Show, don’t tell.

Skill Files

Extend capabilities without writing code.

→ Explore

A SKILL.md file teaches an agent a reusable workflow — no code required. Drop it in .claude/skills/ and invoke it by name.

markdown--- name: code-review description: Review a PR for quality --- ## Steps 1. Read the diff with `gh pr diff` 2. Check for: - Missing error handling - Untested code paths - Security concerns 3. Post review via `gh pr review` ## Constraints - Never approve without reading tests - Flag any secrets in the diff

Tip: Invoke with /code-review in the chat. Skills compose — one skill can call another.

Sub-agent Orchestration

Spawn specialized agents for complex tasks.

→ Explore

Break complex tasks into smaller pieces and dispatch specialized sub-agents for each. A dispatcher routes by risk tier and capability, then merges the results.

pythonasync def dispatch(task): plan = await planner.decompose(task) # Route sub-tasks to specialists jobs = [] for step in plan.steps: agent = pick_agent(step.domain) jobs.append(agent.run(step)) results = await asyncio.gather(*jobs) # Merge & validate return merge_outputs(results)

Tip: Keep each sub-agent focused on one domain. The dispatcher handles coordination and conflict resolution.
