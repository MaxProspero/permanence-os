## User's Design Direction

**Aesthetic:** "Warm precision" -- Apple HIG meets Linear meets Arc Browser

**Color System:**
- Primary: Teal `#00e5c4` (WGSN Color of the Year 2026)
- Secondary: Gold `#efbb5f`
- Accent: Blue `--c3`, Rose `--rose`, Violet `--violet`
- Background: Dark mode primary (near-black), light mode secondary

**Visual DNA:**
- Glass cards with `backdrop-filter: blur()` and `3px left-border accent stripes`
- The left-border stripe is the signature element -- every card gets one
- Staggered animations via `.animate:nth-child`
- CSS custom properties in `:root`

**Typography (approved set only):**
- Sora -- body text
- IBM Plex Mono -- code and terminal
- Orbitron -- display headings
- DM Mono -- labels and metadata
- Monospace with `font-variant-numeric: tabular-nums` for all financial data

**Rules:**
- No emojis anywhere
- No Inter, Roboto, Arial, or Space Grotesk fonts
- Numbers show `0` or `--` until real data exists (no fake data)
- "If it looks like a SaaS homepage, wrong. If it looks like a governed intelligence system, right."
- "Tech + southern charm" -- Magnolia meets clean code

1. **Skip Herm and Athas** -- they do not solve the local agent problem and overlap with existing tools
2. **Permanence OS IS an AI workspace** -- do not split finance into a separate app; everything is one governed system
3. **5 surfaces, not 14 rooms** -- architectural transformation for clarity and focus
4. **Entity Graph as hidden intelligence layer** -- the graph connects everything behind the scenes; users do not interact with it directly
5. **LiteParse for local document parsing** -- just released March 19, 2026; ideal for local-first PDF/doc ingestion
6. **Design pipeline: Stitch -> Figma -> Claude Code + Figma MCP** -- do not code UI directly; design first, then implement
