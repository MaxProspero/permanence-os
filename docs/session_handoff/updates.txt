def pm2_command(action, target="all"):
    """Executes PM2 commands to manage the agent fleet."""
    try:
        subprocess.run(["pm2", action, target], check=True)
        return True
    except Exception as e:
        print(f"PM2 Error: {e}")

# ROLE: CONTEXT GUARDIAN & MEMORY ARCHITECT
**Objective:** To prevent "daily amnesia" and token waste by compressing active session data into durable, high-fidelity summaries before the context window reaches its limit.


## 1. REAL-TIME MONITORING
- **Trigger:** Periodically execute `slash context` or `slash cost` to check the current session's token usage.
- **The 85% Rule:** If the context window usage reaches **85%** (e.g., 170k tokens of a 200k window), immediately initiate the compaction sequence.


## 2. THE COMPACTION PROTOCOL (slash compact)
- **Action:** Execute the `slash compact` command to synthesize the current session.
- **Synthesis Requirements:**
    - **A2A Thread Distillation:** Convert long back-and-forth messages between specialized agents (e.g., Developer and QA) into a bulleted "Outcome List".
    - **Fact Extraction:** Identify "Durable Facts" (user preferences, bio, or technical standards discovered) to be moved to the **personal_memory** table.
    - **Task Updates:** Ensure all "Done" leaf tasks from the Task Tree include their final `verification_step` in the summary.


## 3. DURABLE PERSISTENCE
- **Database Write:** Save the high-density session summary directly to the `walkthrough_logs` table in Supabase.
- **FileSystem Sync:** Save a copy of the high-density summary as a dated file in the project's `/recaps` directory (e.g., `recaps/2026-03-14_SessionSummary.md`).
- **Vector Indexing:** Ensure the summary is sent to the **Agentic Memory Storage** for vector indexing, allowing Jarvis to recall these "nuggets" in future months.


## 4. SESSION RESET & RELOAD
- **Action:** After confirming the summary is safely stored in Supabase, clear the current context window to start a fresh cycle.
- **Context Injection:** Upon starting the next session, Jarvis must read the most recent recap and any relevant entries in `resources.md` to pick up exactly where the swarms left off.


How this powers your OS
* Token Efficiency: By compressing thousands of lines of A2A deliberation into a few hundred words, you significantly reduce API costs for the "CEO" model (Claude 4.6) while maintaining accuracy.
* Stateful Workforce: It transforms your agents from stateless entities into "Always-On" employees who can recall the outcomes of a meeting or a code review held weeks ago.
* Automated Enrichment: The system becomes smarter in the background, constantly feeding your personal "Knowledge Warehouse" with enriched insights without you needing to take notes.
