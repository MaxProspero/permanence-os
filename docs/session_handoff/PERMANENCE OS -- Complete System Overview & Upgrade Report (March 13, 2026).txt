WHAT IS PERMANENCE OS
Permanence OS is a governed personal intelligence operating system built by Payton Hicks, a full-time founder who completed BS and MS degrees in Finance at the University of Arkansas. The project operates under the brand names Permanence OS and Ophtxn, hosted in the private GitHub repository github.com/MaxProspero/permanence-os.
The Problem It Solves
Most AI-assisted development tools give agents unlimited autonomy with no memory, no governance, and no persistent context between sessions. Every conversation starts from scratch. There is no approval process for destructive actions, no spending controls, no audit trail, and no way for the human operator to maintain true authority over what the AI does.
What Permanence OS Does Differently
Permanence OS treats AI agents as governed employees, not autonomous actors. Every significant action passes through approval gates. All API calls are logged with provenance. Spending is gated and auditable. Memory persists across sessions through a layered architecture of chronicles, episodic logs, working memory, and zero-point state. The human operator (Payton) is always the final authority.
Key differentiators:
* Governed execution: Canon-based approval gates, compliance checks, and a secret scanner prevent unauthorized actions.
* Persistent memory: Chronicle system, episodic JSONL logs, working memory, and zero-point state survive across sessions.
* Spending controls: Budget guardrails, daily caps, timed approvals, and cost-recovery planning for API spend.
* Audit-first: Every API call logged, all actions append-only, provenance on every mutation.
* Local-first: Runs entirely on the operator's machine. No cloud dependency for core operations.
* Multi-model routing: Routes tasks to Claude, OpenAI, XAI, or Ollama based on cost, capability, and task type.

THE THREE-LAYER ARCHITECTURE
Permanence OS uses a three-layer context architecture that governs how Claude agents interact with the codebase. This architecture reduces hallucination, keeps agents focused, and eliminates the problem of agents reading irrelevant files.
Layer 1: CLAUDE.md (The Map)
The root-level CLAUDE.md file is the project map. Every agent session reads this first. It defines:
* Project identity (owner, repo, status)
* The floor plan: every top-level directory, what it contains, and when to look there
* Live service URLs and ports
* All 14 site pages with file names and purposes
* Naming conventions for commits, branches, and documents
* Global code standards (Python, HTML/CSS, fonts, colors)
* The "never touch" list (hard rules that must not be violated)
* Routing instructions (how agents should approach tasks)
* Success definition (pytest passing, secret scanner clean, pages rendering correctly)
Layer 2: context.md (Routing Tables)
Per-directory context.md files provide scoped instructions for each workspace. These files tell agents:
* What files exist in this directory
* What tasks are relevant here
* Which files to read for specific task types
* Which files to skip
Context files exist in: site/foundation/, agents/, core/, scripts/, and other workspace directories. This layer dramatically reduces prompt size and keeps agents from wandering into unrelated code.
Layer 3: Workspaces
VS Code workspace configurations (.vscode/) and specialized directories (docs/design/, docs/workflows/) bundle related files with their own rules and templates. This layer provides:
* IDE-level file grouping
* Task-specific workflow templates
* Design rules as a standalone reference document
Why This Matters
Traditional AI coding tools dump the entire codebase into context or rely on the AI to figure out what is relevant. The three-layer architecture means agents start with a map, drill into the relevant workspace, and only read files they actually need. This produces faster, more accurate work with fewer errors.

General Design Rules
* NO emojis anywhere -- use text labels, CSS shapes, or SVG icons
* Always use CSS custom properties, never hardcoded hex in component styles
* All images must have alt text
* Page title format: PageName | Permanence OS
* Meta theme-color: #0a1118
* Staggered animations via .animate:nth-child with fadeUp keyframes
* Film grain overlay on body::before at 0.03 opacity
* Corner bracket decorations using ::before / ::after pseudo-elements
* Menubar: fixed position, z-index: 1000, rgba(10,18,30,0.95) background with backdrop blur
