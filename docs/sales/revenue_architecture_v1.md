# Revenue Architecture v1

## Objective
Turn Permanence OS into a multi-stream business while keeping governance strict.

## Layers
1. Delivery system (your private app stack): command center, automations, agents.
2. Demand system (public web layer): offer pages, proof, intake, CTA.
3. Revenue control system (this layer): pipeline, KPI scorecard, weekly targets.

## Active Streams
1. Done-for-you Foundation setups.
2. Monthly optimization retainers.
3. Custom automation add-on builds.

## Building Streams
1. Templates/playbooks.
2. Hosted app subscription.

## Weekly Operating Rule
- Keep one locked offer and one locked CTA.
- Run money loop daily.
- Update pipeline every day.
- Review scorecard weekly.
