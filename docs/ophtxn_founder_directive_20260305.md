# Ophtxn Founder Directive (March 5, 2026)

## Intent
Build Ophtxn as a trusted personal intelligence system: conversational, loyal, practical, and always improving under governance.

## Hard Constraints
- No autonomous spending.
- No paid plan upgrades or purchases without explicit human approval.
- Prefer free/local/open-source paths first when capability is comparable.
- Human remains final authority for money, publishing, and irreversible actions.

## Product Behavior
- Telegram remains primary day-to-day interface.
- User can send direct system-improvement requests from Telegram to a terminal queue.
- Assistant supports both:
  - practical execution commands
  - natural long-form conversation and strategic thinking

## Team Architecture Hypothesis
- Quality over quantity: compact elite team behavior over large uncontrolled swarms.
- Role archetypes:
  - Orchestrator (point guard): routes work and preserves priorities.
  - Builder (scorer): ships implementation changes.
  - Researcher (scout): finds options and evidence.
  - Critic/Governor (defense): catches risk and policy violations.
  - Memory/Compression lead (playbook): distills patterns into reusable systems.
- Initial target: 3-10 tightly integrated agent roles.

## Research Department Structure
- Keep one senior meta-research function that improves researcher quality.
- Start small with broad researchers + one meta-researcher.
- Expand specialization only when evidence shows bottlenecks.

## Platform Direction
- Phase 1: personal system (Ophtxn for founder).
- Phase 2: guided app for individuals to build their own governed assistant.
- Phase 3: corporate mode for coordinated teams with policy controls.

## Technology Position
- OpenClaw can be used as one interface/tooling layer, not the system identity.
- Maintain provider abstraction so Ophtxn is not tied to one model vendor or UI shell.
- Build for cross-device continuity (desktop + phone + tablet), with governance preserved across surfaces.
