# Security Policy

## Reporting a Vulnerability
Please report security issues privately. Do not open public issues for vulnerabilities.

Send details to: **pbhicksfinn@gmail.com**

Include:
- Description of the issue
- Reproduction steps
- Impact assessment
- Suggested mitigation (if any)

## Supported Versions
We currently support the latest tagged release only.

## Data Handling
- Never commit secrets or personal data.
- Tool outputs and logs are treated as sensitive.

## Scope
Security reports should focus on:
- Unauthorized execution paths
- Bypass of Compliance Gate
- Canon integrity violations
- Leakage of secrets from logs or outputs
